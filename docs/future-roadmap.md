## Phase 1: v1.9.x（2026年 Q2）— 使いやすさの向上

日常的な操作をより快適にするための改善に焦点を当てます。

### ユーザー向け機能

- [ ] **イベント検索** — サイドパネル内でタイトルや説明文をキーワード検索し、該当するイベントをハイライト表示
- [ ] **イベントフィルタリング** — カレンダー別（Google / ローカル / 繰り返し）のイベント表示切替をサイドパネルから直接操作
- [ ] **クイック追加ボタン** — フローティングの「+」ボタンによるイベント簡易作成（タイムラインのドラッグ操作が不要に）
- [ ] **バッジ表示** — 拡張機能アイコンに当日のイベント数をバッジ表示（`chrome.action.setBadgeText`）
- [ ] **通知サウンド** — リマインダー通知にサウンドオプションを追加

### 内部改善

- [ ] `src/services/` ディレクトリの導入 — `background.js` の責務を `CalendarService`、`AuthService` 等に分離
- [ ] `chrome.storage.session` キャッシュ層 — 頻繁にアクセスするデータの読み取り効率を向上

---

## Phase 2: v2.0.0（2026年 Q3）— 視野の拡大

日次ビューを超えた新しい表示形式と、Googleカレンダーへの書き込み機能を導入するメジャーバージョンアップです。

### ユーザー向け機能

- [ ] **週表示（Week View）** — 複数日にまたがるタイムラインを表示する新コンポーネント
  - ヘッダーにビュー切替トグルを追加
  - `EventLayoutManager` を複数カラム表示に対応
  - 日付範囲によるイベント一括取得
- [ ] **Googleカレンダーへの書き込み** — イベントの作成・編集機能（OAuth2スコープの拡張）
- [ ] **イベント説明文プレビュー** — イベントカードのホバーまたは展開で説明文を表示
- [ ] **タイムブロッキング** — 「集中時間」「空き時間」を視覚的にマークし、予定のない時間帯を有効活用

### 内部改善

- [ ] **テストスイートの導入** — Jest / Vitest による自動テスト（優先対象: `time-utils.js`、`EventLayoutManager`、`StorageHelper`）
- [ ] **サービス層の完成** — ビジネスロジックの `src/services/` への完全移行

## Phase 3: v2.1〜v2.3（2026年 Q4）— 世界への展開

国際化の拡充とアクセシビリティの強化により、より多くのユーザーに利用していただける拡張機能を目指します。

### v2.1: 多言語化 & アクセシビリティ

- [ ] **スペイン語（es）、フランス語（fr）、ドイツ語（de）** の追加
- [ ] **フォントサイズ・テキスト設定** — アクセシビリティ向上のための表示カスタマイズ

### v2.2: データ連携の強化

- [ ] **エクスポート機能** — ローカルイベントを iCal / CSV 形式で出力
- [ ] **タイムゾーンサポート** — 複数タイムゾーンの表示対応（`time-utils.js` のリファクタリング）

### v2.3: アジア言語対応

- [ ] **韓国語（ko）、中国語（zh）** の追加
- [ ] `locale-utils.js` の拡張（日付順序、時刻表記の文化的対応）

### 内部改善

- [ ] **パフォーマンス最適化** — 週表示での大量イベント対応（仮想スクロール等）
- [ ] **CI/CD パイプライン** の構築

---

## Phase 4: v3.0（2027年 Q1-Q2）— プラットフォームの進化

マルチプロバイダー対応とデータ分析機能により、SideTimeTable をスケジュール管理の中心ツールへと進化させます。

### ユーザー向け機能

- [ ] **マルチカレンダープロバイダー** — Outlook / Microsoft 365、CalDAV との連携
- [ ] **カレンダー分析・インサイト** — 1日・1週間の時間配分、会議負荷の可視化
- [ ] **Google Tasks 連携** — タスクとイベントの統合表示
- [ ] **オフラインモード** — Service Worker キャッシュと IndexedDB によるオフライン閲覧、再接続時の同期

### 内部改善

- [ ] **プロバイダー抽象化レイヤー** — `CalendarProvider` インターフェースによる統一的なカレンダーアクセス
- [ ] **IndexedDB 統合** — ローカルストレージの大容量化とオフライン対応

## 将来の検討事項

スケジュール未定ですが、検討中のアイデアです。

- **Slack / Teams ステータス同期** — カレンダーの予定に基づくステータス自動更新

- **カスタムイベントテンプレート** — よく使う予定のテンプレート化
- **印刷対応** — 1日・1週間のスケジュール印刷ビュー
